A

Axios API & Architecture

Axios is a promise-based HTTP client that works in both browser and Node.js environments. Its architecture is built around the concept of instances—configurable, reusable request factories that centralise behaviour across your application.

Core Design Principles

  • Promise-based — Native Promise API with async/await support; no callback hell.
  • Instance-centric — Create pre-configured instances via axios.create() rather than mutating the global axios object.
  • Interceptor pipeline — Request and response middleware that transform data, attach headers, or handle errors uniformly.
  • Automatic transforms — JSON request/response body parsing without manual JSON.stringify() or response.json().
  • Browser/Node dual-target — Uses XMLHttpRequest in browsers and http module in Node.js transparently.

The Request Config Object

Every Axios request accepts a config object controlling its behaviour:

interface AxiosRequestConfig {
  url?: string;
  method?: 'get' | 'post' | 'put' | 'patch' | 'delete' | 'head' | 'options';
  baseURL?: string;
  headers?: Record<string, string>;
  params?: Record<string, any>;
  data?: any;
  timeout?: number;
  timeoutErrorMessage?: string;
  withCredentials?: boolean;
  auth?: { username: string; password: string };
  responseType?: 'arraybuffer' | 'blob' | 'document' | 'json' | 'text' | 'stream';
  responseEncoding?: string;
  xsrfCookieName?: string;
  xsrfHeaderName?: string;
  onUploadProgress?: (progressEvent: ProgressEvent) => void;
  onDownloadProgress?: (progressEvent: ProgressEvent) => void;
  validateStatus?: (status: number) => boolean;
  maxRedirects?: number;
  decompress?: boolean;
}
Architecture Tip

Never use the global axios directly in production applications. Always create dedicated instances per API domain (e.g., authApi, dataApi) to prevent configuration leakage between modules.

Cancellation & AbortController

Request cancellation prevents race conditions in React components and reduces unnecessary server load. Axios supports the standard AbortController API (v0.22.0+), replacing the deprecated CancelToken.

React Component Cleanup Pattern

useEffect(() => {
  const controller = new AbortController();

  apiClient.get('/dashboard', { signal: controller.signal })
    .then(response => setData(response.data))
    .catch(err => {
      if (axios.isCancel(err)) {
        console.log('Request cancelled:', err.message);
      } else {
        setError(err);
      }
    });

  return () => controller.abort('Component unmounted');
}, []);

Debounce Search with Cancellation

const useSearch = (query: string) => {
  const [results, setResults] = useState([]);
  const controllerRef = useRef<AbortController | null>(null);

  useEffect(() => {
    if (!query) return;

    controllerRef.current?.abort();
    controllerRef.current = new AbortController();

    apiClient.get('/search', {
      params: { q: query },
      signal: controllerRef.current.signal,
    }).then(r => setResults(r.data));

    return () => controllerRef.current?.abort();
  }, [query]);

  return results;
};

Flask: Detecting Cancelled Requests

@app.route('/api/long-task')
def long_task():
    try:
        for i in range(10):
            time.sleep(1)
            # Check if client disconnected (WSGI may raise ClientDisconnected)
        return jsonify({"done": True})
    except GeneratorExit:
        app.logger.info("Client disconnected mid-request")
        raise
Warning

AbortController only prevents the client from processing the response. The server may still complete the request. For truly interruptible operations, implement server-side cancellation tokens or use WebSockets.

Error Handling & Retry Logic

Robust error handling distinguishes prototype code from production systems. Axios errors carry rich metadata through the AxiosError class.

AxiosError Structure

interface AxiosError<T = unknown> extends Error {
  config: AxiosRequestConfig;
  code?: string;              // 'ECONNABORTED', 'ERR_NETWORK', etc.
  request?: XMLHttpRequest;   // raw request object
  response?: AxiosResponse<T>; // response if server replied
  isAxiosError: boolean;
  toJSON: () => object;
}

Typed Error Handler

class ApiError extends Error {
  constructor(
    public status: number,
    public code: string,
    message: string,
    public errors?: Record<string, string[]>
  ) { super(message); }
}

const handleError = (error: unknown): never => {
  if (axios.isAxiosError(error)) {
    if (error.response) {
      const { status, data } = error.response;
      throw new ApiError(status, data.code, data.message, data.errors);
    }
    if (error.request) {
      throw new ApiError(0, 'NETWORK_ERROR', 'No response received');
    }
    throw new ApiError(0, 'REQUEST_FAILED', error.message);
  }
  throw error;
};

Exponential Backoff Retry

const fetchWithRetry = async <T>(
  request: () => Promise<T>,
  maxRetries = 3,
  baseDelay = 300
): Promise<T> => {
  let lastError: unknown;

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await request();
    } catch (err) {
      lastError = err;
      if (!axios.isAxiosError(err) || (err.response?.status ?? 0) < 500) throw err;

      const delay = baseDelay * Math.pow(2, attempt);
      await new Promise(r => setTimeout(r, delay + Math.random() * 100));
    }
  }
  throw lastError;
};

// Usage
const data = await fetchWithRetry(() => api.get('/critical-data'));
Retry Strategy

Only retry idempotent operations (GET, PUT, DELETE) and 5xx errors. Never blindly retry POST requests—use idempotency keys if retry is necessary.

GET Requests & React Hooks

GET requests form the majority of API traffic. Proper hook-based patterns prevent request waterfalls, duplicate fetches, and stale closures.

useApi Hook Pattern

interface UseApiOptions<T> {
  url: string;
  params?: Record<string, unknown>;
  enabled?: boolean;
  onSuccess?: (data: T) => void;
  onError?: (error: ApiError) => void;
}

function useApi<T>(options: UseApiOptions<T>) {
  const [state, setState] = useState<{
    data: T | null;
    error: ApiError | null;
    isLoading: boolean;
  }>({ data: null, error: null, isLoading: options.enabled !== false });

  useEffect(() => {
    if (options.enabled === false) return;

    const controller = new AbortController();
    setState(s => ({ ...s, isLoading: true, error: null }));

    api.get<T>(options.url, {
      params: options.params,
      signal: controller.signal,
    })
    .then(res => {
      setState({ data: res.data, error: null, isLoading: false });
      options.onSuccess?.(res.data);
    })
    .catch(err => {
      if (axios.isCancel(err)) return;
      const apiErr = normalizeError(err);
      setState({ data: null, error: apiErr, isLoading: false });
      options.onError?.(apiErr);
    });

    return () => controller.abort();
  }, [options.url, JSON.stringify(options.params), options.enabled]);

  return state;
}

Headers & Authentication Tokens

Header management is the backbone of authenticated API communication. Centralised header injection via interceptors eliminates repetitive auth code.

Token Refresh Pattern

let isRefreshing = false;
let refreshSubscribers: ((token: string) => void)[] = [];

api.interceptors.response.use(
  (response) => response,
  async (error: AxiosError) => {
    const originalRequest = error.config!;

    if (error.response?.status === 401 && !originalRequest._retry) {
      if (isRefreshing) {
        return new Promise(resolve => {
          refreshSubscribers.push(token => {
            originalRequest.headers['Authorization'] = `Bearer ${token}`;
            resolve(api(originalRequest));
          });
        });
      }

      originalRequest._retry = true;
      isRefreshing = true;

      try {
        const rs = await axios.post(`${API_URL}/auth/refresh`, {
          refreshToken: getRefreshToken(),
        });
        const { accessToken } = rs.data;
        setAccessToken(accessToken);

        refreshSubscribers.forEach(cb => cb(accessToken));
        refreshSubscribers = [];

        originalRequest.headers['Authorization'] = `Bearer ${accessToken}`;
        return api(originalRequest);
      } finally {
        isRefreshing = false;
      }
    }
    return Promise.reject(error);
  }
);

Interceptors Deep Dive

Interceptors form a middleware pipeline around every request and response. They execute in FIFO order for requests and LIFO order for responses.

Execution Order

Request:  Interceptor A -> Interceptor B -> Server
Response: Server -> Interceptor B -> Interceptor A

Complete Interceptor Stack

// 1. Request: inject auth token
api.interceptors.request.use(config => {
  const token = getAccessToken();
  if (token) config.headers.Authorization = `Bearer ${token}`;
  config.headers['X-Request-ID'] = crypto.randomUUID();
  config.metadata = { startTime: Date.now() };
  return config;
});

// 2. Request: logging (dev only)
if (import.meta.env.DEV) {
  api.interceptors.request.use(config => {
    console.log(`[API] ${config.method?.toUpperCase()} ${config.url}`);
    return config;
  });
}

// 3. Response: normalize errors
api.interceptors.response.use(
  response => {
    // Log timing
    const duration = Date.now() - (response.config.metadata?.startTime ?? 0);
    response.headers['x-response-time'] = `${duration}ms`;
    return response;
  },
  async error => {
    // Circuit breaker pattern
    if (error.response?.status === 503) {
      await circuitBreaker.recordFailure();
    }
    return Promise.reject(error);
  }
);

// Eject interceptors when needed (e.g., logout)
const authInterceptorId = api.interceptors.request.use(/* ... */);
api.interceptors.request.eject(authInterceptorId);

Interceptor ID Management

Store interceptor IDs when you need dynamic ejection (e.g., switching accounts):

const interceptorMap = new Map<string, number>();

export const registerInterceptor = (name: string, interceptor: any) => {
  const id = api.interceptors.request.use(interceptor);
  interceptorMap.set(name, id);
};

export const unregisterInterceptor = (name: string) => {
  const id = interceptorMap.get(name);
  if (id !== undefined) {
    api.interceptors.request.eject(id);
    interceptorMap.delete(name);
  }
};
Interceptor Best Practice

Always return config from request interceptors and response from response handlers. Forgetting the return causes silent request failures that are difficult to debug.

JSON Transforms & Serialisation

Axios automatically serialises request bodies to JSON and parses JSON responses. Override transformRequest and transformResponse for custom formats.

Custom Date Parsing

// Parse ISO date strings to Date objects automatically
api.defaults.transformResponse = [
  ...(axios.defaults.transformResponse as AxiosTransformer[]),
  (data) => {
    if (typeof data !== 'object' || data === null) return data;
    return deepMap(data, (value) => {
      if (typeof value === 'string' && ISO_DATE_REGEX.test(value)) {
        return new Date(value);
      }
      return value;
    });
  }
];

Snake Case to Camel Case

import { camelCase, isPlainObject, mapKeys, mapValues } from 'lodash';

const transformKeys = (obj: unknown, transformer: (s: string) => string): unknown => {
  if (Array.isArray(obj)) return obj.map(v => transformKeys(v, transformer));
  if (isPlainObject(obj)) {
    return mapValues(
      mapKeys(obj as Record<string, unknown>, (_, k) => transformer(k)),
      v => transformKeys(v, transformer)
    );
  }
  return obj;
};

// Flask sends snake_case; React expects camelCase
api.defaults.transformResponse = [
  ...(axios.defaults.transformResponse as AxiosTransformer[]),
  data => transformKeys(data, camelCase)
];

Flask: JSON Encoder for Custom Types

from flask.json import JSONEncoder
from datetime import datetime, date
from decimal import Decimal

class CustomJSONEncoder(JSONEncoder):
    def default(self, obj):
        if isinstance(obj, datetime):
            return obj.isoformat()
        if isinstance(obj, date):
            return obj.isoformat()
        if isinstance(obj, Decimal):
            return float(obj)
        if isinstance(obj, set):
            return list(obj)
        return super().default(obj)

app.json_encoder = CustomJSONEncoder

Loading States & React Suspense

Loading state management directly impacts perceived performance. Combine Axios with React Suspense for declarative loading boundaries.

Suspense-Compatible Data Fetching

// Create a resource wrapper that Suspense understands
function createResource<T>(promise: Promise<T>) {
  let status: 'pending' | 'success' | 'error' = 'pending';
  let result: T;
  let error: unknown;

  const suspender = promise.then(
    r => { status = 'success'; result = r; },
    e => { status = 'error'; error = e; }
  );

  return {
    read(): T {
      if (status === 'pending') throw suspender;
      if (status === 'error') throw error;
      return result;
    }
  };
}

// Usage in component
const UserProfile: React.FC<{ userId: string }> = ({ userId }) => {
  const resource = useMemo(
    () => createResource(api.get(`/users/${userId}`).then(r => r.data)),
    [userId]
  );

  const user = resource.read(); // Suspends until resolved
  return <div>{user.name}</div>;
};

// Wrap with Suspense boundary
<Suspense fallback={<Skeleton height={200} />}>
  <UserProfile userId={id} />
</Suspense>

Optimistic UI with Loading States

const useMutation = <TData, TVariables>(
  mutationFn: (vars: TVariables) => Promise<TData>,
  options?: { onSuccess?: () => void }
) => {
  const [state, setState] = useState<{ isLoading: boolean; error: Error | null }>(
    { isLoading: false, error: null }
  );

  const mutate = async (variables: TVariables) => {
    setState({ isLoading: true, error: null });
    try {
      const result = await mutationFn(variables);
      setState({ isLoading: false, error: null });
      options?.onSuccess?.();
      return result;
    } catch (error) {
      setState({ isLoading: false, error: error as Error });
      throw error;
    }
  };

  return { mutate, ...state };
};

OAuth 2.0 / JWT Authentication Flow

Modern React-Flask architectures use JWT access/refresh token pairs stored in httpOnly cookies or memory, with Axios managing the silent refresh cycle.

Architecture Overview

React SPA          Axios           Flask Backend
   |                 |                    |
   | -- login ------>| -- POST /auth ------>|
   |                 |                    | verify, set httpOnly refresh cookie
   |                 | <-- { accessToken }-|
   |                 | store in memory    |
   | <-- success --- |                    |
   |                 |                    |
   | -- API call --->| -- with Bearer ---->|
   |                 |                    | verify JWT

const updateUserOptimistic = async (userId: string, updates: Partial<User>) => {
  const previousUsers = queryClient.getQueryData<User[]>(['users']);

  // Optimistically update cache
  queryClient.setQueryData(['users'], (old: User[] | undefined) =>
    old?.map(u => u.id === userId ? { ...u, ...updates } : u) ?? []
  );

  try {
    const { data } = await usersApi.update(userId, updates);
    return data;
  } catch (error) {
    // Rollback on failure
    queryClient.setQueryData(['users'], previousUsers);
    throw error;
  }
};

Retry with Exponential Backoff & Jitter

const sleep = (ms: number) => new Promise(r => setTimeout(r, ms));

const retryWithBackoff = async <T>(
  fn: () => Promise<T>,
  options: { maxRetries?: number; baseDelay?: number; maxDelay?: number } = {}
): Promise<T> => {
  const { maxRetries = 3, baseDelay = 300, maxDelay = 10_000 } = options;

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error) {
      const isRetryable = axios.isAxiosError(error) &&
        (!error.response || error.response.status >= 500 ||
         error.code === 'ECONNABORTED' || error.code === 'ERR_NETWORK');

      if (!isRetryable || attempt === maxRetries) throw error;

      const exponential = baseDelay * Math.pow(2, attempt);
      const jitter = Math.random() * exponential * 0.5;
      const delay = Math.min(exponential + jitter, maxDelay);

      console.warn(`Retry ${attempt + 1}/${maxRetries} after ${delay.toFixed(0)}ms`);
      await sleep(delay);
    }
  }
  throw new Error('Unreachable');
};

const validateResponse = <T>(
  schema: z.ZodSchema<T>,
  response: unknown
): T => {
  const result = schema.safeParse(response);
  if (!result.success) {
    console.error('API response validation failed:', result.error.flatten());
    throw new Error(`Invalid response: ${result.error.message}`);
  }
  return result.data;
};

Zero-Dependency Alternatives

While Axios remains the industry standard, understanding alternatives helps make informed architectural decisions and appreciate Axios's trade-offs.

LibrarySizeStrengthsWhen to Choose

Fetch API 0 KB (builtin) Native, streaming, zero bundle Simple requests, modern browsers, size-critical apps

Fetch vs Axios: Quick Comparison

// Fetch: manual error handling, no timeout built-in
const fetchUsers = async () => {
  const controller = new AbortController();
  const timeout = setTimeout(() => controller.abort(), 10000);

  try {
    const res = await fetch('/api/users', {
      signal: controller.signal,
      headers: { 'Accept': 'application/json' },
    });
    clearTimeout(timeout);

    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    return await res.json();
  } catch (err) {
    clearTimeout(timeout);
    throw err;
  }
};

// Axios: interceptors, automatic JSON, timeout, cancel built-in
const getUsers = () => api.get('/users', { timeout: 10000 });

When to Migrate Away from Axios

  • Bundle size critical — Mobile apps where every KB matters
  • Streaming responses — Server-Sent Events, large file downloads (native Fetch/Streams API is more capable)
  • Edge runtimes — Cloudflare Workers, Deno Deploy where XHR is unavailable
  • Simplicity sufficient — Apps with only a few simple GET/POST calls
Decision Framework

Start with Axios for any non-trivial project. Its interceptor ecosystem, mature TypeScript support, and extensive documentation reduce development time. Only consider alternatives if bundle size or specific Fetch capabilities become blockers.
