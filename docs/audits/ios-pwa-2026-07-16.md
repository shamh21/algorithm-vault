# AlgVault iOS PWA audit — 2026-07-16

## Architecture

- Flask/Jinja application served on Vercel through `server.py`.
- Public and authenticated routes are Flask blueprints; private execution remains server-authoritative.
- Shared application shell: `templates/base.html`, `static/css/app.css`, `static/css/public.css`, `static/css/algvault-theme.css`, and `static/js/app-shell.js`.
- PWA resources: `static/manifest.json` and `static/js/sw.js`, exposed as `/manifest.json` and `/sw.js`.
- Separate Next.js admin PWA under `admin-pwa`, validated by the existing CI workflow.

## Material findings

- Production contained mobile-drawer fixes on an ephemeral Vercel branch that were absent from `main`.
- The public site rendered fabricated balances, provider counts, latency, readiness, provider integrations, and activity history.
- Public feature-category controls appeared interactive but had no behavior.
- The `main` drawer remained `inert` when opened and did not preserve page position or contain/restore focus.
- The service worker skipped waiting immediately while the UI expected an explicit update/reload flow.
- Static service-worker caches had no entry cap.
- CI still referenced the retired `app.algvault.com` origin.
- Vercel runtime history showed earlier template/route failures and expected fail-closed wallet-generation blockers.

## Implementation

- Rebuilt public marketing output around shared, compliant templates with no public balance, invented provider activity, execution history, or optimistic readiness claims.
- Consolidated semantic black/crimson/purple design tokens, restrained elevation, readable contrast, tabular numerals, modern viewport units, safe-area spacing, and 44px touch targets.
- Hardened mobile navigation with `inert` management, focus containment/restoration, Escape and tap-outside dismissal, exact scroll restoration, and landscape handling.
- Added a dismissible iPhone Safari install-help surface that stays hidden in standalone mode.
- Added visual-viewport tracking for keyboard-safe mobile layouts and form-button restoration after back-forward cache navigation.
- Changed the manifest to support both orientations and retained valid maskable icons and protected-route shortcuts.
- Changed service-worker updates to explicit activation, forced critical-shell refresh, bounded static caches, network-only auth/API handling, and truthful offline messaging.
- Added regression tests for public claims, navigation behavior, safe areas, input sizing, manifest metadata, and service-worker security/update behavior.
- Extended CI with JavaScript syntax checks, manifest validation, route-map verification, and the new regression tests.

## Compliance posture

- Sensitive operations remain server-authoritative.
- Protected actions are not represented as successful before server confirmation.
- Offline state never treats cached data as execution success.
- Simulated, pending, restricted, rejected, failed, and completed states remain semantically distinct.
- No investment-advice or guaranteed-return language was added.
